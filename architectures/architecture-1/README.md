# Architecture 1 – Centralized Identity

## Business Scenario

As ACME grows, multiple enterprise applications require employees to maintain separate accounts and credentials.

The company decides to introduce a centralized Identity Provider to simplify authentication, improve security and automate user provisioning across applications.

## Business Problem

How can an organization centralize authentication while maintaining automatic user provisioning across enterprise applications?

## Learning Objectives

- Deploy an enterprise Identity Provider.
- Configure Single Sign-On using OpenID Connect.
- Implement automated identity provisioning.
- Design a reusable provisioning engine.

## Technologies

- Keycloak
- OpenID Connect
- OAuth2
- Docker
- PostgreSQL
- Python
- REST APIs
- Gitea

## Architecture Overview

- Identity Provider (Keycloak)
- Single Sign-On
- Automated Identity Provisioning
- Modular Provisioning Engine

## Learning Outcomes

After completing this architecture, learners will understand how enterprise organizations centralize authentication, implement Single Sign-On and automate identity lifecycle operations using modern IAM technologies.
